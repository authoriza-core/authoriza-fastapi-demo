# FastAPI OAuth 2.0 / OpenID Connect Demo (интеграция Python FastAPI с Авториза)

## Назначение проекта

Этот репозиторий — пример приложения на Python/FastAPI, которое реализует авторизацию веб-приложения к сервису Авториза.

**Основной функционал:**
- Авторизация пользователя через внешний OIDC-провайдер
- Сохранение и восстановление сессии
- Автоматическое обновление access_token до истечения срока действия
- Выход из системы с очисткой всех данных

---

## Стек и используемые библиотеки

| Компонент | Технология | Назначение |
|---|---|---|
| **Веб-фреймворк** | FastAPI | REST API, маршрутизация |
| **OAuth-клиент** | Authlib | OIDC Discovery/PKCE |
| **ASGI-сервер** | Uvicorn | Запуск приложения |
| **Шаблонизатор** | Jinja2 | HTML |
| **Токены** | python-jose | Декодирование JWT payload |
| **Конфигурация** | python-dotenv | Загрузка переменных окружения |

## Настройка приложения в Авторизе

Для работы приложения необходима регистрация клиента в **Авторизе**.

### Шаг 1. Регистрация приложения

1. Перейдите в панель управления Авторизы
2. Создайте новый проект и приложение в нём
3. Укажите **Redirect URI**: `http://127.0.0.1:8000/callback`
4. Тип приложения: **Confidential**

### Шаг 2. Получение credentials

Скопируйте из конфигурации приложения:
- **client_id**
- **client_secret**

### Шаг 3. Настройка переменных окружения

Создайте файл `.env` в корне проекта:

```env
CLIENT_ID=***
CLIENT_SECRET=***
REDIRECT_URI=http://127.0.0.1:8000/callback
SECRET_KEY=***
```

`SECRET_KEY` — случайная строка минимум 32 символа (используется для подписи сессий).

## Проверка основных сценариев

### 1. Успешная авторизация

```
1. Откройте http://127.0.0.1:8000
2. Нажмите кнопку "Войти" (страница /login)
3. Введите учётные данные Авторизы
4. В случае успешной авторизации произойдёт редирект на главную страницу
```

**Ожидаемый результат:** на главной странице отображается информация о пользователе и о токенах в декодированном виде.

### 2. Восстановление сессии

```
1. Авторизуйтесь в приложении
2. Остановите сервер (Ctrl+C)
3. Запустите сервер снова
4. Вернитесь на главную страницу
```

**Ожидаемый результат:** сессия восстановлена автоматически (без повторного логина).

### 3. Автоматическое обновление токенов

```
1. Авторизуйтесь в приложении
2. Дождитесь момента истечения access_token
3. Выполните любой запрос к приложению (или переход на главную страницу)
```

**Ожидаемый результат:** токены автоматически обновлены.

## Структура репозитория

| Файл | Назначение |
|---|---|
| `main.py` | Точка входа, конфигурация FastAPI, middleware сессий |
| `auth.py` | Маршруты: `/`, `/login`, `/callback`, `/logout`, `/status`, `/refresh` |
| `oauth_client.py` | Настройка OAuth-клиента через Authlib |
| `session_service.py` | Восстановление сессии |
| `token_service.py` | Проверка срока действия и обновление токенов |
| `auth_storage.py` | Логика и функционал, связанные с JSON (сохранение/загрузка/удаление данных авторизации) |
| `templates/index.html` | HTML-шаблон страницы |
| `requirements.txt` | Зависимости проекта |
| `.env` | Конфиденциальные настройки (credentials) |
